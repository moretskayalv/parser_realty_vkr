# parser_realty_vkr

Репозиторий содержит код и результаты подготовки датасета по объектам недвижимости в Москве для выпускной квалификационной работы.

Проект реализует полный пайплайн: сбор объявлений, гео-обогащение, добавление инфраструктурных признаков, очистку данных, построение географических кластеров, генерацию синтетических объектов и последующую проверку качества данных.

> Папка `old_files` не относится к актуальной версии пайплайна.  
> Графики и интерактивные карты используются как визуальные результаты, но не являются входом для основного пайплайна.

## 3. Описание этапов

### Этап 1. Парсинг объявлений

**Скрипт:** `scr/parser/parser_etagi_onemore.py`

Скрипт собирает объявления с сайта недвижимости.

Что делает:

- читает поисковые группы из `search_group.csv`;
- проходит по страницам выдачи;
- собирает ссылки на карточки объявлений;
- открывает каждую карточку;
- извлекает характеристики объекта;
- получает координаты;
- сохраняет сырые данные.

Основные собираемые признаки:

- цена;
- площадь;
- количество комнат;
- этаж и этажность дома;
- адрес;
- координаты;
- ближайшее метро и время до метро;
- площадь кухни;
- ремонт;
- материал стен;
- год постройки;
- высота потолков;
- лифт;
- двор, парковка, детская площадка;
- ссылка на объявление.

Результаты:

| Файл | Описание |
|---|---|
| `outputs/links.csv` | Собранные ссылки на объявления |
| `outputs/result.csv` | Сырые данные после парсинга карточек |

---

### Этап 2. Базовое гео-обогащение

**Скрипт:** `scr/enrichment/enrich_base.py`

Скрипт дополняет исходные объявления географическими признаками.

Что делает:

- восстанавливает адрес по координатам, если адрес отсутствует;
- определяет ближайшую станцию метро по координатам;
- считает расстояние до ближайшего метро;
- считает примерное время пешком до метро;
- считает расстояние до центра Москвы;
- считает расстояние до МКАД.

Результат:

| Файл | Описание |
|---|---|
| `outputs/result_final.csv` | Данные после базового гео-обогащения |

Ключевые признаки:

- `address_source` — источник адреса;
- `metro_source` — источник информации о метро;
- `metro_distance_km` — расстояние до ближайшего метро;
- `metro_walk_min` — примерное время пешком до метро;
- `distance_to_center_km` — расстояние до центра Москвы;
- `distance_to_mkad_km` — расстояние до МКАД.

---

### Этап 3. Корректировка расстояния до МКАД

**Скрипт:** `scr/enrichment/fix_mkad_distance.py`

На этом этапе расстояние до МКАД пересчитывается более точно по геометрии МКАД.

Что делает:

- использует файл `config/mkad_osm.geojson`;
- строит геометрию МКАД;
- пересчитывает расстояние от каждого объекта недвижимости до ближайшей точки МКАД.

Результат:

| Файл | Описание |
|---|---|
| `outputs/result_final_mkad_fixed.csv` | Данные с исправленным расстоянием до МКАД |

---

### Этап 4. Обогащение расстояниями до инфраструктуры

**Скрипт:** `scr/enrichment/add_poi_distances_local.py`

Скрипт добавляет признаки доступности городской инфраструктуры.

Что делает:

- использует локальный файл `config/moscow_poi.csv`;
- выделяет основные типы объектов инфраструктуры;
- для каждой квартиры считает расстояние до ближайшего объекта каждого типа.

Добавляемые признаки:

- `distance_to_school_km` — расстояние до ближайшей школы;
- `distance_to_kindergarten_km` — расстояние до ближайшего детского сада;
- `distance_to_clinic_km` — расстояние до ближайшей поликлиники/клиники;
- `distance_to_mall_km` — расстояние до ближайшего торгового центра.

Результат:

| Файл | Описание |
|---|---|
| `outputs/result_final_geo_poi.csv` | Данные после добавления расстояний до инфраструктуры |

---

### Этап 5. Очистка данных и feature engineering

**Скрипт:** `scr/clean data/dataset_clean.py`

Скрипт очищает датасет и готовит его к анализу и моделированию.

Что делает:

- нормализует названия колонок;
- приводит числовые признаки к числовому типу;
- удаляет строки с критичными пропусками;
- удаляет дубликаты;
- фильтрует выбросы;
- заполняет часть пропусков;
- нормализует категориальные признаки;
- создаёт дополнительные признаки.

Добавляемые признаки:

- `price_per_m2` — цена за квадратный метр;
- `floor_ratio` — отношение этажа квартиры к этажности дома;
- `house_age` — возраст дома;
- `has_metro_near` — бинарный признак близости метро.

Результаты:

| Файл | Описание |
|---|---|
| `outputs/dataset_clean_before_encoding.csv` | Очищенный датасет до финального ML-кодирования |
| `outputs/dataset_ready_for_ml-2.csv` | Подготовленная версия датасета для дальнейшего ML-этапа |

## 4. Новая логика кластеризации

### Этап 6. Построение геокластеров для генерации синтетики

**Скрипт:** `scr/enrichment/cluster_map.py`

В актуальной версии проекта кластеризация используется не как отдельный районный признак, а как технический географический слой для более корректной генерации синтетических объектов.

Что делает:

- загружает `dataset_clean_before_encoding.csv`;
- берёт только координаты `latitude` и `longitude`;
- стандартизирует координаты;
- строит 25 географических кластеров с помощью `MiniBatchKMeans`;
- добавляет колонку `geo_generation_cluster`;
- считает сводку по каждому кластеру;
- сохраняет статичную и интерактивную карту кластеров.

Почему кластеризация сделана только по координатам:

- синтетические объекты должны генерироваться рядом с реальными объектами;
- нельзя смешивать квартиры из разных частей Москвы;
- геокластеры ограничивают генерацию локальными пространственными зонами;
- после генерации координаты немного смещаются, но остаются внутри своего кластера.

Результаты:

| Файл | Описание |
|---|---|
| `outputs/dataset_with_geo_generation_clusters.csv` | Очищенный датасет с колонкой `geo_generation_cluster` |
| `outputs/geo_generation_cluster_summary.csv` | Сводка по геокластерам |
| `geo_generation_clusters.png` | PNG-карта геокластеров |
| `geo_generation_clusters_interactive.html.zip` | Архив с интерактивной HTML-картой |

## 5. Новая логика генерации синтетических данных

### Этап 7. Генерация синтетических объектов внутри геокластеров

**Скрипт:** `scr/enrichment/geo_generation_cluster.py`

Скрипт создаёт синтетические объекты на основе реальных квартир внутри одного и того же географического кластера.

Важное отличие от старой версии: синтетика создаётся не простым копированием строк с шумом, а через интерполяцию между двумя реальными объектами внутри одного геокластера.

Что делает:

- загружает `dataset_with_geo_generation_clusters.csv`;
- делит данные на train и test;
- test-выборку не изменяет;
- внутри каждого геокластера выбирает пары реальных объектов;
- создаёт синтетическую строку как смесь двух объектов;
- координаты синтетического объекта интерполируются между координатами двух родителей;
- к координатам добавляется небольшой шум около 35 метров;
- координаты ограничиваются границами своего геокластера;
- непрерывные признаки интерполируются и слегка шумятся;
- дискретные признаки округляются;
- категориальные признаки берутся от одного из родительских объектов;
- цена пересчитывается согласованно через `price_per_m2 * area_total`;
- геозависимые признаки временно очищаются, чтобы пересчитать их заново после генерации.

Почему геозависимые признаки очищаются:

- после смещения координат старые расстояния до метро, центра, МКАД, школ, детских садов, клиник и торговых центров становятся некорректными;
- поэтому такие поля обнуляются;
- затем они пересчитываются отдельным скриптом.

Служебные признаки:

- `is_synthetic` — 0 для реальных объектов, 1 для синтетических;
- `generation_method` — способ генерации;
- `parent_geo_cluster` — геокластер, внутри которого был создан объект.

Результаты:

| Файл | Описание |
|---|---|
| `outputs/train_original_before_synthetic.csv` | Исходная train-выборка до генерации синтетики |
| `outputs/test_original_not_augmented.csv` | Test-выборка без синтетики |
| `outputs/synthetic_objects_need_geo_recalculation.csv` | Только синтетические объекты, которым нужно пересчитать геопризнаки |
| `outputs/train_augmented_need_geo_recalculation.csv` | Train-выборка: реальные + синтетические объекты до пересчёта геопризнаков |
| `outputs/geo_cluster_generation_report.csv` | Отчёт по количеству объектов в геокластерах |

### Этап 8. Пересчёт геопризнаков для синтетических объектов

**Скрипт:** `scr/agumentation/syntetic_object_geo_generation.py`

Скрипт повторно обогащает синтетические объекты после генерации новых координат.

Что делает:

- загружает `synthetic_objects_need_geo_recalculation.csv`;
- удаляет строки без координат;
- заново определяет ближайшее метро;
- пересчитывает расстояние до метро;
- пересчитывает примерное время пешком до метро;
- пересчитывает расстояние до центра Москвы;
- пересчитывает расстояние до МКАД по `mkad_osm.geojson`;
- пересчитывает расстояния до POI из `moscow_poi.csv`;
- при необходимости восстанавливает адрес через reverse geocoding;
- сохраняет синтетические объекты с актуальными географическими признаками.

Результат:

| Файл | Описание |
|---|---|
| `synthetic_objects_geo_enriched.csv` | Синтетические объекты после повторного гео-обогащения |

## 9. Краткое описание

В проекте был реализован полный цикл подготовки данных по недвижимости в Москве.

Сначала объявления были собраны с сайта недвижимости. Затем данные были дополнены географическими признаками: адресом, ближайшим метро, расстоянием до метро, расстоянием до центра Москвы и расстоянием до МКАД. После этого были добавлены признаки доступности инфраструктуры: расстояния до школ, детских садов, клиник и торговых центров.

Далее датасет был очищен: удалены дубликаты, критичные пропуски и выбросы, числовые признаки были приведены к корректным типам, а также были добавлены новые признаки, например цена за квадратный метр, возраст дома и отношение этажа к этажности дома.

В актуальной версии проекта отдельно построены географические кластеры по координатам объектов. Эти кластеры используются для генерации синтетических объектов: новые строки создаются через интерполяцию между реальными объектами внутри одного геокластера. Это позволяет не смешивать квартиры из разных районов Москвы и сохранять пространственную реалистичность данных.

После генерации синтетических координат все геозависимые признаки для синтетических объектов пересчитываются заново: ближайшее метро, расстояние до метро, расстояние до центра, расстояние до МКАД и расстояния до объектов инфраструктуры. Это нужно, потому что после изменения координат старые расстояния уже не соответствуют новому местоположению объекта.

Итоговые данные и промежуточные результаты находятся в папке `outputs`, а EDA-ноутбуки для проверки качества данных находятся в папке `EDA`.
