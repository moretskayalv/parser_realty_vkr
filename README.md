# parser_realty_vkr

Репозиторий содержит код и результаты подготовки датасета по объектам недвижимости в Москве для выпускной квалификационной работы.

Основная логика проекта — полный пайплайн от сбора объявлений с сайта до получения очищенного и обогащённого датасета, а также создания синтетической обучающей выборки для сравнения моделей.

> Папки `old_files` и `plots` являются вспомогательными/архивными и не используются в основном пайплайне.

---

```text

   scr/parser/parser_etagi_onemore.py

   scr/enrichment/add_poi_distances_local.py

   scr/agumentation/data_agumentation.py
```

## 3. Описание этапов

### Этап 1. Парсинг объявлений

**Скрипт:** `scr/parser/parser_etagi_onemore.py`

Скрипт собирает объявления с сайта недвижимости по группам поисковых ссылок.

Что делает:

- читает поисковые группы из `search_group.csv`;
- проходит по страницам выдачи;
- собирает ссылки на карточки объявлений;
- открывает каждую карточку через Playwright;
- раскрывает блок характеристик;
- получает координаты через блок карты;
- извлекает основные признаки объекта.

Основные собираемые признаки:

- цена;
- общая площадь;
- количество комнат;
- этаж и этажность дома;
- адрес;
- широта и долгота;
- метро и время пешком до метро;
- площадь кухни;
- ремонт;
- материал стен;
- год постройки;
- высота потолков;
- наличие лифта;
- двор, парковка, детская площадка;
- ссылка на объявление.

Результаты:

- `outputs/links.csv` — собранные ссылки на объявления;
- `outputs/result.csv` — сырые данные, полученные после парсинга карточек.

---

### Этап 2. Базовое гео-обогащение

**Скрипт:** `scr/enrichment/enrich_base.py`

Скрипт дополняет данные географическими признаками.

Что делает:

- если адрес отсутствует, восстанавливает его по координатам через reverse geocoding;
- пересчитывает ближайшую станцию метро по координатам;
- добавляет расстояние до ближайшей станции метро;
- добавляет примерное время пешком до метро;
- считает расстояние до центра Москвы;
- считает расстояние до МКАД.

Результат:

- `outputs/result_final.csv`

Добавляемые признаки:

- `address_source` — источник адреса: исходный парсинг или геокодирование;
- `metro_source` — источник метро;
- `metro_distance_km` — расстояние до ближайшего метро;
- `distance_to_center_km` — расстояние до центра Москвы;
- `distance_to_mkad_km` — расстояние до МКАД.

---

### Этап 3. Корректировка расстояния до МКАД

**Скрипт:** `scr/enrichment/fix_mkad_distance.py`

На этом этапе расстояние до МКАД пересчитывается более корректно по геометрии МКАД из OpenStreetMap.

Что делает:

- загружает локальный файл геометрии МКАД `mkad_osm.geojson`, если он уже есть;
- если файла нет, скачивает геометрию МКАД через Overpass API;
- строит геометрию МКАД;
- пересчитывает расстояние от каждого объекта до ближайшей точки МКАД.

Результат:

- `outputs/result_final_mkad_fixed.csv`

---

### Этап 4. Обогащение расстояниями до инфраструктуры

**Скрипт:** `scr/enrichment/add_poi_distances_local.py`

Скрипт добавляет расстояния до ближайших объектов городской инфраструктуры.

Что делает:

- использует локальный файл `moscow_poi.csv`, если он уже есть;
- если файла нет, скачивает POI по Москве через Overpass API;
- выделяет несколько типов объектов:
  - школы;
  - детские сады;
  - поликлиники и больницы;
  - торговые центры;
- для каждого объекта недвижимости считает расстояние до ближайшего объекта каждого типа.

Результат:

- `outputs/result_final_geo_poi.csv`

Добавляемые признаки:

- `distance_to_school_km`;
- `distance_to_kindergarten_km`;
- `distance_to_clinic_km`;
- `distance_to_mall_km`.

---

### Этап 5. Очистка данных и подготовка признаков

**Скрипт:** `scr/clean data/dataset_clean.py`

Скрипт очищает обогащённый датасет перед дальнейшим анализом и моделированием.

Что делает:

- загружает `result_final_geo_poi.csv`;
- нормализует названия колонок;
- приводит числовые признаки к числовому типу;
- удаляет строки с критичными пропусками;
- удаляет дубликаты;
- фильтрует выбросы по цене, площади, этажам, году постройки и расстояниям;
- заполняет часть пропусков медианами;
- нормализует категориальные признаки;
- добавляет новые признаки.

Добавляемые признаки:

- `price_per_m2` — цена за квадратный метр;
- `floor_ratio` — отношение этажа квартиры к этажности дома;
- `house_age` — возраст дома;
- `has_metro_near` — бинарный признак близости метро.

Результат:

- `outputs/dataset_clean_before_encoding.csv`

---

### Этап 6. Добавление пространственных районов / кластеров

**Скрипт:** `scr/enrichment/district_cluster.py`

После очистки к данным добавляется пространственный кластер, который используется как приближённый аналог района.

Что делает:

- берёт очищенный датасет `dataset_clean_before_encoding.csv`;
- использует пространственные признаки:
  - широту;
  - долготу;
  - расстояние до центра;
  - расстояние до МКАД;
  - расстояние до метро;
- стандартизирует признаки;
- применяет KMeans;
- создаёт колонку `district_cluster`.

Результат:

- `outputs/dataset_with_spatial_districts.csv`

---

### Этап 7. Генерация синтетических данных

**Скрипт:** `scr/agumentation/data_agumentation.py`

Скрипт создаёт синтетическую обучающую выборку на основе пространственных кластеров.

Важно: тестовая выборка не аугментируется. Это нужно, чтобы сравнение моделей было честным.

Что делает:

- загружает `dataset_with_spatial_districts.csv`;
- делит данные на train и test;
- для train-части создаёт синтетические копии внутри каждого пространственного кластера;
- добавляет небольшой шум к координатам;
- добавляет небольшой шум к расстояниям и непрерывным числовым признакам;
- не изменяет целевую переменную `price`;
- добавляет служебные признаки `is_synthetic` и `augmentation_copy`;
- формирует отчёт по количеству синтетических строк в каждом кластере.

Результаты:

- `outputs/train_original.csv` — исходная train-выборка без аугментации;
- `outputs/test_original_not_augmented.csv` — тестовая выборка без аугментации;
- `outputs/train_cluster_augmented.csv` — train-выборка с синтетическими строками;
- `outputs/cluster_augmentation_report.csv` — отчёт по аугментации.

## 4. EDA и проверка данных

### `EDA/Quality_EDA_real_estate_pipeline.ipynb`

Ноутбук используется для анализа качества данных после очистки и обогащения.

В нём можно проверить:

- размер датасета;
- пропуски;
- распределения числовых признаков;
- выбросы;
- качество географических признаков;
- распределение цен, площадей, расстояний и других переменных.

### `EDA/Post_Augmentation_Validation_EDA-2.ipynb`

Ноутбук используется для проверки данных после генерации синтетической выборки.

В нём можно сравнить:

- исходный train и аугментированный train;
- распределения признаков до и после аугментации;
- реалистичность синтетических наблюдений;
- сохранение структуры пространственных кластеров;
- пригодность синтетических данных для дальнейшего сравнения моделей.

## 7. Краткое описание

В проекте был реализован полный цикл подготовки данных по недвижимости:

1. Сначала были собраны объявления с сайта недвижимости.
2. Затем данные были дополнены адресами, ближайшим метро, расстоянием до метро, центра Москвы и МКАД.
3. После этого были добавлены признаки доступности городской инфраструктуры: школ, детских садов, поликлиник и торговых центров.
4. Далее датасет был очищен: удалены дубликаты, критичные пропуски и выбросы, а также добавлены новые признаки.
5. Затем были построены пространственные кластеры, которые используются как приближённые районы.
6. На основе этих кластеров была создана синтетическая обучающая выборка.
7. Для проверки качества данных и сравнения исходной и синтетической выборок были подготовлены EDA-ноутбуки.

Итоговые данные для дальнейшего моделирования находятся в папке `outputs`.
